## Запуск AWS Academy Learner Lab

### Крок 1.1 — Вхід до платформи
1. Перейдіть на сайт [awsacademy.instructure.com](https://awsacademy.instructure.com).
2. Введіть ваш **Email** та **Password**, натисніть **Login**.

### Крок 1.2 — Відкриття лабораторії
1. На головній сторінці оберіть ваш **курс**.
2. У лівому меню натисніть **Modules**.
3. Знайдіть та відкрийте **Learner Lab — Foundational Services**.
4. Натисніть кнопку **Launch AWS Academy Learner Lab**.

### Крок 1.3 — Запуск Lab-сесії
1. У вікні Learner Lab натисніть кнопку **Start Lab** у верхньому лівому куті.

   > ⏳ Зачекайте, поки індикатор поруч з написом **AWS** зміниться з червоного 🔴 на зелений 🟢. Це займає ~1–2 хвилини.

2. Натисніть на зелений напис **AWS** або кнопку **Open Console**, щоб відкрити AWS Management Console.

   > ⚠️ **Важливо:** AWS Academy Learner Lab надає тимчасові облікові дані. Сесія має обмежений час (зазвичай 4 години). Не закривайте вкладку Learner Lab під час роботи.

## Створення S3 Bucket

**Amazon S3 (Simple Storage Service)** — об'єктне сховище для зберігання будь-яких файлів.

### Крок 2.1 — Перехід до S3
1. У рядку пошуку вгорі AWS Console введіть **S3**.
2. Натисніть на **S3** у результатах пошуку.

### Крок 2.2 — Створення нового bucket
1. Натисніть кнопку **Create bucket**.

### Крок 2.3 — Налаштування bucket

**Блок "General configuration":**
| Поле | Значення |
|---|---|
| Bucket name | `my-lab-bucket-[ваше-ім'я]` (лише малі літери, цифри, дефіс) |
| AWS Region | `us-east-1` (або поточний регіон) |

> ⚠️ Ім'я bucket має бути **унікальним глобально** по всьому AWS. Рекомендується додати ваше ім'я або дату, наприклад: `my-lab-bucket-john-2024`.

**Блок "Object Ownership":**
- Залиште **ACLs disabled** (рекомендовано).

**Блок "Block Public Access settings":**
- За замовчуванням увімкнено **Block all public access** — залиште це налаштування для безпеки.

**Блок "Versioning":**
- Для навчальних цілей можна залишити **Disable**.

**Блок "Default encryption":**
- Залиште **Server-side encryption with Amazon S3 managed keys (SSE-S3)**.

### Крок 2.4 — Завершення створення
1. Прокрутіть сторінку донизу.
2. Натисніть кнопку **Create bucket**.
3. Ви побачите повідомлення: ✅ *Successfully created bucket "my-lab-bucket-..."*

## Завантаження файлів у S3

### Крок 3.1 — Відкриття bucket
1. У списку bucket-ів натисніть на щойно створений **my-lab-bucket-...**.

### Крок 3.2 — Завантаження файлу
1. Натисніть кнопку **Upload**.
2. Натисніть **Add files** та оберіть файл на вашому комп'ютері  
   (наприклад, текстовий файл `hello.txt` або будь-який інший).
3. Натисніть **Upload** внизу сторінки.
4. Дочекайтеся повідомлення ✅ *Upload succeeded*.
5. Натисніть **Close**.

### Крок 3.3 — Перевірка завантаженого файлу
1. У вкладці **Objects** ви побачите завантажений файл.
2. Запам'ятайте або запишіть **ім'я файлу** — воно знадобиться пізніше.

## Створення EC2 Instance

**Amazon EC2 (Elastic Compute Cloud)** — хмарний сервер (віртуальна машина).

### Крок 4.1 — Перехід до EC2
1. У рядку пошуку вгорі AWS Console введіть **EC2**.
2. Натисніть на **EC2** у результатах.
3. На Dashboard натисніть кнопку **Launch instance**.

### Крок 4.2 — Налаштування instance

**Name and tags:**
| Поле | Значення |
|---|---|
| Name | `my-lab-server` |

**Application and OS Images (AMI):**
1. Оберіть **Amazon Linux** (зазвичай виділено за замовчуванням).
2. Переконайтеся, що обрано **Amazon Linux 2023 AMI** або **Amazon Linux 2 AMI**.
3. Архітектура: **64-bit (x86)**.

**Instance type:**
| Поле | Значення |
|---|---|
| Instance type | `t2.micro` (позначено як **Free tier eligible**) |

**Key pair (login):**
1. Натисніть **Create new key pair**.
2. Заповніть поля:

   | Поле | Значення |
   |---|---|
   | Key pair name | `my-lab-key` |
   | Key pair type | **RSA** |
   | Private key file format | **.pem** (для Linux/Mac) або **.ppk** (для Windows/PuTTY) |

3. Натисніть **Create key pair**.
4. Файл ключа (`my-lab-key.pem`) автоматично **завантажиться на ваш комп'ютер**.

   > ⚠️ **Збережіть цей файл надійно!** Без нього ви не зможете підключитися до сервера. Повторно завантажити його неможливо.

**Network settings:**
1. Натисніть **Edit**.
2. Переконайтеся, що **Auto-assign public IP** встановлено в **Enable**.
3. У блоці **Firewall (security groups)** оберіть **Create security group**.
4. Назва: `my-lab-sg`.
5. Переконайтеся, що є правило: **Type: SSH, Port: 22, Source: 0.0.0.0/0**.

   > 💡 Для продакшн-середовища рекомендується обмежувати Source до вашої IP-адреси, але для навчання `0.0.0.0/0` прийнятно.

**Configure storage:**
| Поле | Значення |
|---|---|
| Root volume | `8 GiB` (за замовчуванням) |
| Volume type | `gp3` |

**IAM instance profile (важливо для доступу до S3!):**
1. Розгорніть блок **Advanced details**.
2. Знайдіть поле **IAM instance profile**.
3. Із випадного списку оберіть **LabInstanceProfile** (або схожу роль).

   > ℹ️ Ця роль дозволить EC2 отримати доступ до S3 без введення ключів доступу. В AWS Academy ця роль вже налаштована.

### Крок 4.3 — Запуск instance
1. Перегляньте блок **Summary** праворуч.
2. Натисніть **Launch instance**.
3. Ви побачите повідомлення ✅ *Successfully initiated launch of instance*.
4. Натисніть **View all instances**.

### Крок 4.4 — Очікування запуску
1. Знайдіть ваш instance `my-lab-server` у списку.
2. Зачекайте, поки у стовпці **Instance state** з'явиться **running** 🟢, а у **Status check** — **2/2 checks passed**.

   > ⏳ Зазвичай це займає 1–3 хвилини.

3. Клікніть на instance та у нижній панелі знайдіть і скопіюйте **Public IPv4 address** — він знадобиться для підключення.

## Підключення до EC2

### Варіант А — Через EC2 Instance Connect (рекомендовано для Learner Lab)

Це найпростіший спосіб, який не потребує SSH-клієнта.

1. У списку Instances оберіть ваш `my-lab-server` (поставте галочку).
2. Натисніть кнопку **Connect** вгорі.
3. Оберіть вкладку **EC2 Instance Connect**.
4. Username залиште `ec2-user`.
5. Натисніть **Connect**.
6. Відкриється термінальне вікно прямо у браузері. ✅

### Варіант Б — Через SSH (Linux / macOS)

1. Відкрийте термінал.
2. Перейдіть до папки, куди завантажився ключ:
   ```bash
   cd ~/Downloads
   ```
3. Встановіть правильні права на файл ключа:
   ```bash
   chmod 400 my-lab-key.pem
   ```
4. Підключіться до instance (замініть `<PUBLIC_IP>` на вашу IP-адресу):
   ```bash
   ssh -i "my-lab-key.pem" ec2-user@<PUBLIC_IP>
   ```
5. При запиті *Are you sure you want to continue connecting?* введіть **yes**.

### Варіант В — Через PuTTY (Windows)

1. Відкрийте **PuTTY**.
2. У полі **Host Name** введіть: `ec2-user@<PUBLIC_IP>`.
3. У лівому меню: **Connection → SSH → Auth → Credentials**.
4. У поле **Private key file** завантажте ваш файл `.ppk`.
5. Натисніть **Open**, потім **Accept**.

## Копіювання файлів з S3 на EC2

Після підключення до EC2 через термінал виконайте наступні кроки.

### Крок 6.1 — Перевірка підключення
У терміналі ви маєте бачити запрошення виду:
```
[ec2-user@ip-172-xx-xx-xx ~]$
```

### Крок 6.2 — Перевірка доступу до S3
Виконайте команду для перегляду вмісту вашого bucket:
```bash
aws s3 ls s3://my-lab-bucket-[ваше-ім'я]/
```

Ви маєте побачити список файлів, що зберігаються у bucket. Якщо команда виконалась успішно — IAM-роль налаштована правильно. ✅

### Крок 6.3 — Копіювання одного файлу з S3 на EC2
Використовуйте команду `aws s3 cp`:
```bash
aws s3 cp s3://my-lab-bucket-[ваше-ім'я]/hello.txt ./hello.txt
```

**Формат команди:**
```
aws s3 cp s3://<bucket-name>/<шлях-до-файлу> <локальний-шлях>
```

Приклад результату:
```
download: s3://my-lab-bucket-john-2024/hello.txt to ./hello.txt
```

### Крок 6.4 — Перевірка завантаженого файлу
```bash
ls -la
```
Ви побачите файл `hello.txt` у поточній директорії.

Переглянути вміст файлу:
```bash
cat hello.txt
```

### Крок 6.5 — Копіювання всього вмісту bucket на EC2
Щоб завантажити **всі файли** з bucket рекурсивно:
```bash
aws s3 cp s3://my-lab-bucket-[ваше-ім'я]/ ./ --recursive
```

### Крок 6.6 — Синхронізація папки з bucket (альтернатива)
Команда `sync` завантажує лише нові або змінені файли:
```bash
aws s3 sync s3://my-lab-bucket-[ваше-ім'я]/ ./s3-files/
```

### Крок 6.7 — Копіювання файлу з EC2 назад до S3
Якщо потрібно завантажити файл **з EC2 у S3**:
```bash
# Спочатку створимо тестовий файл
echo "Hello from EC2!" > output.txt

# Завантажуємо на S3
aws s3 cp output.txt s3://my-lab-bucket-[ваше-ім'я]/output.txt
```

## Довідник команд AWS CLI для S3

| Команда | Опис |
|---|---|
| `aws s3 ls` | Переглянути всі bucket |
| `aws s3 ls s3://bucket-name/` | Переглянути вміст bucket |
| `aws s3 cp s3://bucket/file.txt ./` | Завантажити файл з S3 |
| `aws s3 cp ./file.txt s3://bucket/` | Завантажити файл у S3 |
| `aws s3 cp s3://bucket/ ./ --recursive` | Завантажити всі файли з S3 |
| `aws s3 sync s3://bucket/ ./folder/` | Синхронізувати bucket з локальною папкою |
| `aws s3 rm s3://bucket/file.txt` | Видалити файл із S3 |
| `aws s3 mb s3://new-bucket-name` | Створити новий bucket |

## ⚠️ Важливі нагадування для Learner Lab

- **Зупиняйте Lab-сесію** після завершення роботи кнопкою **End Lab** — невикористані ресурси витрачають бюджет Labs.
- **Ресурси не зберігаються між сесіями** — EC2 та S3 будуть видалені після завершення сесії (якщо не збережено в межах акаунту).
- **Бюджет Lab** зазвичай становить $50–100. Стежте за індикатором бюджету у вікні Learner Lab.
- **Регіон:** завжди переконуйтеся, що працюєте в **одному регіоні** для EC2 і S3 — за замовчуванням `us-east-1`.
