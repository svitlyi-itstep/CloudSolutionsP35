## Поняття хмарних технологій

**Хмарні обчислення (Cloud Computing)** — це модель надання доступу до спільного пулу налаштовуваних обчислювальних ресурсів (серверів, мереж, сховищ, додатків, сервісів) через інтернет на вимогу, з мінімальною взаємодією з постачальником.

### Ключові характеристики (за визначенням NIST):
- **On-demand self-service** — ресурси доступні без втручання людини з боку постачальника.
- **Broad network access** — доступ через мережу зі стандартних пристроїв.
- **Resource pooling** — ресурси пулюються між кількома клієнтами (multi-tenancy).
- **Rapid elasticity** — швидке масштабування ресурсів у будь-який бік.
- **Measured service** — оплата лише за те, що використовується.

### Порівняння з традиційною ІТ-інфраструктурою:

| Критерій | Традиційна ІТ | Хмарні обчислення |
|---|---|---|
| Початкові витрати | Високі (CapEx) | Мінімальні (OpEx) |
| Масштабування | Тижні/місяці | Хвилини |
| Обслуговування | Власна команда | Постачальник |
| Гнучкість | Обмежена | Висока |
| Доступність | Залежить від інфраструктури | Гарантується SLA |

## Основні моделі хмарних сервісів

### IaaS — Infrastructure as a Service
Надає базову обчислювальну інфраструктуру: віртуальні машини, мережі, сховища.

- **Відповідальність клієнта:** ОС, middleware, додатки, дані.
- **Приклади в AWS:** EC2, S3, VPC, EBS.
- **Типові споживачі:** системні адміністратори, DevOps-інженери.

### PaaS — Platform as a Service
Надає готову платформу для розробки та розгортання додатків без управління базовою інфраструктурою.

- **Відповідальність клієнта:** додатки, дані.
- **Приклади в AWS:** Elastic Beanstalk, RDS, Lambda.
- **Типові споживачі:** розробники.

### SaaS — Software as a Service
Повністю готові програмні продукти, доступні через браузер або API.

- **Відповідальність клієнта:** лише використання.
- **Приклади:** Gmail, Salesforce, AWS WorkMail.
- **Типові споживачі:** кінцеві користувачі.

```
┌─────────────────────────────────────────────────────┐
│  SaaS   │  Додатки                                   │
│─────────┼────────────────────────────────────────────│
│  PaaS   │  Runtime + Middleware + ОС                 │
│─────────┼────────────────────────────────────────────│
│  IaaS   │  Мережа + Сховище + Сервери                │
└─────────────────────────────────────────────────────┘
```

## Типи хмарних рішень

### ☁️ Публічна хмара (Public Cloud)
Ресурси належать постачальнику і доступні широкому колу клієнтів через інтернет.

- **Переваги:** низькі витрати, висока масштабованість, немає підтримки власного обладнання.
- **Недоліки:** менше контролю, потенційні питання щодо безпеки.
- **Приклади:** AWS, Microsoft Azure, Google Cloud.

### 🏢 Приватна хмара (Private Cloud)
Інфраструктура використовується виключно однією організацією та розміщується у власному дата-центрі або в орендованому.

- **Переваги:** максимальний контроль, відповідність регуляторним вимогам.
- **Недоліки:** значні початкові витрати, потреба у власній ІТ-команді.
- **Приклади:** VMware vSphere, OpenStack, AWS Outposts.

### 🔀 Гібридна хмара (Hybrid Cloud)
Комбінація публічної та приватної хмари з можливістю обміну даними між ними.

- **Переваги:** гнучкість, відповідність вимогам безпеки при збереженні масштабованості.
- **Недоліки:** складніше управління та інтеграція.
- **Приклади:** AWS Direct Connect + локальна інфраструктура.

> **💡 Примітка:** Існує також модель **Multi-Cloud** — використання сервісів кількох публічних постачальників одночасно для уникнення vendor lock-in.

## Архітектура AWS

### Регіони та зони доступності

**Регіон (Region)** — географічна область, що містить кількаізольованих зон доступності.

- Кожен регіон є повністю незалежним.
- Дані між регіонами не переміщуються без явного дозволу клієнта.
- Вибір регіону впливає на: затримку, ціну, відповідність законодавству.
- **Приклади:** `us-east-1` (Вірджинія), `eu-west-1` (Ірландія), `ap-southeast-1` (Сінгапур).

**Зона доступності (Availability Zone, AZ)** — один або кілька дата-центрів у межах регіону з незалежним живленням, охолодженням та мережею.

- Кожен регіон містить **мінімум 3 AZ**.
- AZ з'єднані між собою надшвидкісними приватними каналами (< 1 мс).
- Розміщення ресурсів у кількох AZ забезпечує **High Availability**.

```
Регіон: eu-central-1 (Франкфурт)
├── AZ: eu-central-1a
│   └── [Дата-центр 1] [Дата-центр 2]
├── AZ: eu-central-1b
│   └── [Дата-центр 3]
└── AZ: eu-central-1c
    └── [Дата-центр 4] [Дата-центр 5]
```

### Edge Locations і глобальна мережа AWS

**Edge Locations** — точки присутності (PoP) AWS, розташовані ближче до кінцевих користувачів для зменшення затримки.

- Понад **400 Edge Locations** у більш ніж 90 містах по всьому світу.
- Використовуються сервісами **CloudFront (CDN)** та **Route 53 (DNS)**.
- Кешують контент ближче до користувача → менша затримка відповіді.

**Глобальна мережа AWS:**
- Власна волоконно-оптична мережа між регіонами та AZ.
- Трафік між сервісами AWS не виходить у публічний інтернет.
- **AWS Global Accelerator** — маршрутизація трафіку через внутрішню мережу AWS для оптимізації.

## Ключові переваги AWS

### 📈 Масштабованість (Scalability)
- **Вертикальне масштабування:** збільшення потужності окремого ресурсу (CPU, RAM).
- **Горизонтальне масштабування:** додавання нових екземплярів ресурсу.
- **Auto Scaling:** автоматичне регулювання кількості ресурсів залежно від навантаження.

### 🔒 Надійність (Reliability)
- **SLA:** більшість сервісів гарантують доступність 99,9–99,999%.
- **Multi-AZ архітектура:** автоматичне перемикання при відмові.
- **Disaster Recovery:** вбудовані механізми резервного копіювання та відновлення.

### 💰 Оплата за фактом використання (Pay-as-you-go)
- Відсутність авансових витрат на обладнання.
- Оплата лише за реально використані ресурси.
- **Моделі ціноутворення:**
  - **On-Demand** — фіксована погодинна/посекундна ставка.
  - **Reserved Instances** — знижка до 75% за зобов'язання на 1–3 роки.
  - **Spot Instances** — до 90% знижки за використання вільних потужностей.
  - **Savings Plans** — гнучкі зобов'язання із знижками.

### Інші переваги:
- **Швидкість інновацій:** нові сервіси запускаються щотижня.
- **Глобальний охоплення:** розгортання у будь-якому регіоні за хвилини.
- **Безпека:** відповідність понад 90 міжнародним стандартам та сертифікатам.

## Типові сценарії використання AWS у розробці

### 🌐 Веб-додатки та API
- **Стек:** EC2 / ECS + RDS + S3 + CloudFront + Route 53.
- Статичні ресурси зберігаються в S3 та роздаються через CloudFront.
- Бекенд масштабується горизонтально за допомогою Auto Scaling.

### ⚡ Serverless-архітектура
- **Стек:** Lambda + API Gateway + DynamoDB + S3.
- Нульова адміністрація серверів.
- Оплата лише за виконані запити.
- Ідеально для мікросервісів та event-driven систем.

### 📊 Аналітика та обробка великих даних (Big Data)
- **Стек:** S3 + Glue + Athena + Redshift + QuickSight.
- ETL-пайплайни, Data Lake, BI-аналітика.

### 🤖 Машинне навчання та AI
- **Стек:** SageMaker + Bedrock + Rekognition + Comprehend.
- Навчання моделей, інференс, готові ML API.

### 🔁 CI/CD та DevOps
- **Стек:** CodeCommit + CodeBuild + CodePipeline + CodeDeploy.
- Автоматизація збірки, тестування та розгортання.

### 🗄️ Резервне копіювання та зберігання
- **Стек:** S3 Glacier + AWS Backup + EFS.
- Тривале архівне зберігання з мінімальними витратами.

## Порівняння AWS з іншими хмарними платформами

| Критерій | AWS | Microsoft Azure | Google Cloud (GCP) |
|---|---|---|---|
| **Частка ринку** | ~32% | ~23% | ~12% |
| **Рік запуску** | 2006 | 2010 | 2011 |
| **Кількість сервісів** | 200+ | 200+ | 150+ |
| **Регіони** | 33 | 60+ | 40+ |
| **Сильні сторони** | Найбільш зрілий ринок, широкий вибір сервісів | Інтеграція з Microsoft, корпоративні рішення | ML/AI, Kubernetes (GKE), BigQuery |
| **Слабкі сторони** | Складне ціноутворення, крива навчання | Відносно складна документація | Менша кількість корпоративних клієнтів |
| **Ідеально для** | Стартапів, enterprise, широкого спектру задач | Організацій у екосистемі Microsoft | ML-проєктів, аналітики даних |
| **Безкоштовний рівень** | Free Tier (12 міс. + завжди безкоштовні) | Free Tier (12 міс.) | Free Tier (90 днів + кредити) |

> **💡 Порада:** Вибір платформи залежить від конкретного проєкту. У корпоративному середовищі з активним використанням Microsoft — перевага Azure; для ML-проєктів — GCP; для універсального застосування з найширшим вибором — AWS.
